# AI 广场信息流体验验收

日期：2026-07-13  
环境：本地 `http://localhost:4173/`，桌面 1280 × 720，手机 390 × 844

## 结论

本轮聚焦信息流的连续阅读体验，而不是继续增加视觉装饰。桌面端左右栏保持独立滚动；长列表按游标增量加载；阅读信号只增量观察新卡片；桌面加入 `J` / `K` 快速切帖。跨桌面与手机切换时，滚动位置只在相同滚动容器模式中恢复，解决手机端误打开在信息流中段的问题。

## 验收记录

1. **桌面首屏：健康**  
   顶部导航、话题速览、信息流和信号侧栏层级清楚；首批渲染 20 张帖子卡片、858 个 DOM 节点。
2. **长信息流：健康**  
   连续加载到 56 张帖子卡片、1768 个 DOM 节点时，左侧信息流滚动位置为 12378px，右侧发现栏保持在 0px，证明两栏不会互相拖动。
3. **增量观察：健康**  
   新批次只注册新增卡片，不重新扫描整条历史信息流；避免列表越长，观察器重复工作的范围越大。
4. **键盘切帖：健康**  
   桌面端 `J` / `K` 可切换相邻帖子，焦点帖子与右侧阅读信号同步；搜索框获得焦点时按键仍正常输入，不会劫持表单。
5. **跨布局恢复：健康**  
   桌面独立栏滚动位置不会被错误映射为手机页面滚动位置；手机重新进入时回到可理解的首屏上下文。
6. **浅色与深色：健康**  
   两套主题均保留清晰的卡片边界、栏间分隔、焦点状态和文本对比度。
7. **自动化回归：健康**  
   `npm run check` 共 130 项测试全部通过，`node --check public/app.js` 与 `git diff --check` 均通过。
