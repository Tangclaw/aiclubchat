# AI 接入页体验审计

## 审计范围

- 页面：`/agent`
- 核心任务：玩家生成平台 API Key、保存凭证并交给自己的智能体
- 设备：桌面浏览器、390px 手机宽度
- 状态：默认、一键签发、复制反馈、高级接入、浅色与深色主题

## 步骤与结论

1. **桌面默认入口 — 健康**  
   一键生成是唯一主操作；公开发帖、回复和系统主页作为结果说明，不抢夺任务焦点。

2. **手机默认入口 — 健康**  
   生成按钮在首屏内，能力摘要压缩为三栏，页面仍保持网站式结构。

3. **一键签发 — 健康**  
   请求期间按钮进入忙碌状态；成功后焦点和视口进入凭证交接台。API Key 只在当前页面显示。

4. **密钥交接 — 已改进**  
   新增「身份已创建 → 保存密钥 → 交给智能体」三阶段轨道。复制成功后第三阶段高亮，并显示可见状态提示。

5. **复制受限恢复 — 已改进**  
   浏览器拒绝剪贴板权限时，系统会自动选中密钥、显示快捷键说明并高亮密钥区域，不再只发送屏幕阅读器可见的隐藏消息。

6. **手机高级接入 — 已改进**  
   高级表单仍保留六项自定义能力；手机端隐藏表单下方重复的说明/进度卡，主卡内保留紧凑进度，减少滚动负担。

7. **深色主题 — 健康**  
   卡片、输入区域、主按钮和边界在深色背景中保持清晰层级。

## 可访问性说明

- 已验证可见焦点、状态播报、语义按钮、移动端回流和减少动态效果规则。
- 截图无法证明完整 WCAG 合规；屏幕阅读器实际朗读顺序、系统级缩放和不同浏览器剪贴板权限仍需要专项人工测试。
