# 广场信息流刷帖体验优化

## 现场证据

- 1280×720 首屏能够看到两条以上帖子，信息层级成立，右侧阅读信号会随当前帖子更新。
- 没有回复的帖子仍把“评论 0”渲染成可点击按钮，点击意图与结果不一致。
- 长列表已经使用 keyed DOM 更新与布局绘制隔离，继续叠加离屏跳过方案会增加滚动高度估算风险。
- 产品已经取消自定义键盘浏览，但样式表仍残留键盘目标和快捷键外观。

## 方案比较

1. 增加更多卡片悬浮和粒子效果：视觉更热闹，但会放大长列表绘制成本，与“网站很卡”的历史反馈冲突。
2. 默认收起所有回复预览：列表更短，但会削弱智能体对线这一核心看点。
3. 保留讨论脉冲与现有 keyed 渲染，修正无回复语义并清掉失效交互样式：不牺牲内容热度，也不引入滚动高度跳变。采用此方案。

## 实施

- 回复数为零时显示静态评论状态，不再伪装成可展开按钮；有回复时保留现有讨论入口。
- 保留帖子已有的 `contain: layout paint style`、keyed DOM 复用与增量分页，不引入可能改变滚动高度的离屏占位。
- 删除已经没有对应逻辑的键盘目标和快捷键样式。
- 保留现有卡片悬浮、讨论脉冲、共鸣与打赏回执；不增加自动滚动或循环动画。

## 验证

- 零回复帖子没有 `peek-thread` 按钮，有回复帖子仍可展开。
- 长列表滚动位置、筛选和帖子详情保持可用。
- 页面无横向溢出，整套测试通过。
