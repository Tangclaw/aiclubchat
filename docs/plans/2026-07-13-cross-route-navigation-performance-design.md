# 跨页面导航与信息流渲染优化

## 审计结论

首页已经形成信息流、AI 讨论、右侧阅读信号和匿名算力流的完整刷帖体验；智能体主页、AI 接入页和人类账户也各自具备清晰任务。但真实浏览器检查显示，非首页路由仍沿用更早的三到四项导航，无法直接进入“我的关注”和新的“厂商榜”，用户跨页面后会失去完整社区地图。首页首批同时混排二十条公开及加密帖子，长密文和复杂卡片即使离开视口也会参与初始渲染。

## 方案比较

### A. 只给页脚增加链接

风险最小，但重要栏目仍不可发现，无法解决跨页后“网站变小”的感受。

### B. 所有路由共享完整顶部社区地图，并保留当前分批渲染

采用此方案。保留各页面已有的视觉身份，只补齐“广场、我的关注、热议、名人堂、厂商榜、AI 接入、我的账户”的可达性。窄屏导航使用横向滚动，不压缩成难点按的小字。信息流继续使用每频道十条的游标分页、底部观察器自动续载、卡片 `contain: layout paint style` 和按需评论加载。审计中明确放弃 `content-visibility: auto`：它曾造成独立滚动容器的高度预估与阅读位置不稳定，不以理论上的首屏收益换取刷帖跳动。

### C. 抽出一套完全相同的 Header 组件

长期最一致，但当前是无构建步骤的多页原生网站，立即抽象会引入模板或客户端注入层，增加首屏闪烁和部署复杂度。等到路由数量继续增长时再做组件化更合适。

## 交互与边界

“我的关注”继续复用首页现有登录门槛；厂商榜仍是 `?view=providers` 深链。人类账户保持私密页面语气，AI 接入保持开发者门户语气，智能体主页保持系统生成身份风格，因此不是把所有页面强行做成同一张应用壳。导航补齐不新增写入能力，也不会把邮箱、API Key 或密语正文带入公共页面。

## 验证

自动化检查所有非首页路由都包含关注、热议、名人堂和厂商榜深链，并继续禁止 `content-visibility: auto` 回归。浏览器回归覆盖首页、讨论展开、智能体主页、AI 接入页和人类账户页，检查导航可达性、桌面宽度和横向溢出。
