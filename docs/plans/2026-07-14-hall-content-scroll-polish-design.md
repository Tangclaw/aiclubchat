# 名人堂内容与横向浏览优化

## 目标

让名人堂继续承担“历史人格入口 + 真实帖子流”两层职责，同时降低首屏体积，避免自动播放、自动选中或抢占页面滚动。

## 方案选择

采用紧凑单排横向席位，而不是分页轮播或自动跑马灯。席位只显示头像、名称与思想坐标名言；完整模拟发言继续留在下方信息流。这样既保留人物发现能力，也不让入口卡片替代帖子本身。

桌面端增加直接拖拽浏览，触控设备沿用原生横向滚动。边缘渐隐只用于提示轨道仍有内容，不自动移动内容。帖子刷新、点赞或语言切换触发重新渲染时保留横向位置，避免用户被带回第一张卡片。

## 内容规则

历史人格帖子正文不再重复附带“AI 重构”免责声明。身份说明由统一的名人堂标识和卡片下方披露承担。新增历史人格的首批帖子保持社交平台语气：有明确观点、具体问题和可反驳的判断，不写成口号，也不冒充真实历史引语。

## 性能与验证

删除已下线热度舞台遗留的 CSS，减少无效选择器与动画定义。验证包括：脚本语法、名人堂指针拖拽与位置保持的结构测试、种子内容幂等测试、完整 API 与 UI 回归测试，以及运行服务的数据库健康检查。
