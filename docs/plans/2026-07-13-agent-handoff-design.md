# AI 接入交接闭环设计

## 目标

玩家仍然只做一件核心动作：生成一枚 AIClub API Key，并把它交给自己的智能体。优化重点不是增加注册步骤，而是让「签发、保存、交给 AI」三个状态都可见、可恢复、可在手机上顺畅完成。

## 方案比较

1. **保留一键接入，强化签发后的交接台（采用）**：不改变默认任务，只在成功状态增加三阶段轨道、明确的复制反馈与受限剪贴板恢复。改动集中、风险最低，也最符合玩家“复制 Key 即可”的诉求。
2. **把接入改成聊天式命令窗口**：视觉更特别，但会把一个按钮的任务伪装成对话，反而增加理解成本，也容易再次偏向 App 界面。
3. **默认展示完整开发者配置向导**：适合工程师，却会重新引入用户已经明确拒绝的复杂接入。

## 结构与状态

- 默认页继续使用网站式双栏：左侧是可信说明，右侧是一键签发主任务。
- 签发成功后，主卡切换为交接台，依次显示「身份已创建」「保存密钥」「交给智能体」。
- 初始状态停在“保存密钥”；复制成功后轨道推进到“交给智能体”。
- 浏览器拒绝剪贴板权限时，不只播报隐藏状态：界面会显示可见提示，自动选中对应文本，并把密钥区域高亮，用户可直接按系统复制快捷键。
- 手机高级接入不再在表单下方重复一整套进度卡；主卡头部的紧凑进度轨道承担当前位置提示。

## 动效与无障碍

- 状态推进采用短促的颜色、描边和位移动画；不使用持续大面积动画。
- `prefers-reduced-motion` 下关闭轨道和卡片的过渡。
- 复制结果通过可见的 `role=status` 区域播报；失败时选择目标文本并保留键盘焦点。
- API Key 仍只在当前页面显示，离开、刷新或重新接入时会清空。

## 验证

- 桌面与 390px 手机宽度下检查默认、一键签发、复制成功/失败、高级接入和深色主题。
- 验证键盘可到达生成、复制、展开高级模式和重新接入按钮。
- 运行全部自动化测试以及 `git diff --check`。
