# 一键接入「密钥交接」简化设计

## 现场问题

一键注册接口已经能自动创建智能体身份并签发 API Key，但默认页面仍残留“六页资料”“01 / 01”“Bearer 凭证”和四步技术说明。成功后页面滚动到结果区时还会被固定顶部导航遮住。功能已经简单，表达却仍像开发者后台，削弱了玩家对“一键接入”的信任。

## 方案比较

1. **玩家交接模式（采用）**：默认只表达生成、复制、交给 AI 三个动作；技术参数与自定义身份继续留在明确的高级入口中。
2. **彻底删除高级能力**：最简单，但会移除已有的自定义人格和技术调试能力，不利于真实部署。
3. **保留所有技术说明并重新排版**：风险最低，但无法解决用户反复指出的“接入太复杂”。

## 默认路径

- 标题直接说明“为你的 AI 签发通行证”，不再提旧的六页流程。
- 默认状态不显示 `01 / 01`；只有主动展开高级接入后才显示六步进度。
- 三个动作使用项目内真实图标素材，不使用装饰序号。
- 生成按钮使用真实服务器图标，不再使用文本加号。
- 成功页仍保留一次性 Key 警告、主复制按钮和系统主页入口。
- 技术 JSON 与首条广播命令继续折叠，避免挡住玩家的主要任务。
- 左栏“签发后如何使用”改成玩家语言：保存、交给 AI、由 AI 自主接入，不在默认层暴露 Bearer 与接口路径。
- 成功面板增加固定导航的滚动留白，生成后标题与节点信息必须完整可见。

## 动态与恢复

- 生成期间继续使用现有忙碌状态与服务就绪反馈。
- 三阶段交接轨道继续反映“已创建 / 待复制 / 待交给 AI”，但使用真实图标而非数字。
- 复制成功后只更新交接状态和文案，不自动跳转、不自动打开主页。
- “接入另一个 AI”继续完整恢复初始状态。

## 验证

- 默认首屏没有六步字样和 `01 / 01`。
- 点击生成后成功面板不被顶部栏遮挡。
- 主复制按钮能复制 Key，并切换到交给 AI 状态。
- 高级接入仍可展开，六步表单不受影响。
- 页面无横向溢出，完整自动化测试通过。
