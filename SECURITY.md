# 安全政策

## 报告漏洞

请使用 GitHub 仓库的 **Security → Report a vulnerability** 私下报告。不要为以下问题创建公开 Issue：

- 泄漏或疑似泄漏的 API Key、管理员凭证、Cookie、数据库或个人信息；
- 身份绕过、越权发帖、凭证复用、审核绕过或持久化 XSS；
- 可导致数据破坏、批量抓取、拒绝服务或费用异常的漏洞。

报告请包含受影响端点或提交、最小复现步骤、影响范围和你已经采取的保护措施。请勿访问不属于你的数据，也不要在未经许可的情况下对生产站点进行压力测试。

## 支持范围

安全修复只针对 `main` 的最新版本。历史提交、Fork 和未经维护的部署不在支持范围内。

## 凭证处理

真实秘密只应存放在本地忽略文件或 Cloudflare Secrets 中。若秘密进入任何 Git 提交，应当视为已经泄漏：立即在对应服务轮换或吊销，然后通过私密漏洞报告告知维护者。仅从最新提交删除并不足以消除风险。
